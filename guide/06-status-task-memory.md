# 06. 상태, 작업, 메모리 관리

## 상태 확인

가장 기본적인 상태 확인 명령은 아래다.

```text
/status
```

또는 자연어로 물어볼 수도 있다.

```text
현재 상태를 알려줘
현재 시스템 상태를 확인해줘
```

보통 아래 정보를 확인할 수 있다.

- 현재 채널
- chat ID
- model 또는 target
- context 설정
- web 기능 여부
- exec 기능 여부
- 최근 작업 상태

## WebUI 상태 표시

WebUI 에서는 상태가 여러 위치에 표시된다.

- 대시보드: 전체 요약
- 채팅 헤더: 현재 채팅 타깃과 채널
- 상태 요약 줄: 사용자 중심 요약, 막힘 상태, action result
- 세부 정보: 자세한 자동화 결과
- 사이드바 배지: 승인 대기 또는 채널 정보

상태 요약 줄은 여러 카드 대신 현재 대화에 필요한 신호만 한 줄로 읽게 하는 역할이다.

예를 들어 아래 정보가 합쳐질 수 있다.

- 관련 세션 수
- 현재 채널
- 최근 활동 시각

## 작업 상태란

Nanobot 은 단순히 한 번 답하고 끝나는 것이 아니라, 일부 작업 상태를 세션 metadata 로 남긴다.

예시:

- 현재 작업 요약
- 막힌 작업
- 승인 대기
- 최근 자동화 결과
- 선제 업데이트 보류 또는 quiet-hours 상태

이 정보는 WebUI 대시보드와 채팅 화면에서 다시 볼 수 있다.

## reasoning 표시란

현재 WebUI 는 observable reasoning 을 raw chain-of-thought 전체가 아니라, 사용자에게 안전한 상태/요약 중심으로 보여준다.

설정의 `추론 표시 수준` 에 따라 아래처럼 다르게 보일 수 있다.

- 끄기: reasoning 줄을 숨긴다.
- 상태만: `생각 중` 같은 상태만 본다.
- 요약: 답변 앞에 `생각` 한 줄 요약이 남는다.
- 디버그 추적: 더 자세한 trace 를 본다.

최근 작업에서는 새로고침 후에도 assistant 답변 앞에 reasoning 요약 줄이 복원되도록 보강됐다.

## action result 란

일정, 메일 같은 자동화 결과는 보통 compact row 로 먼저 보인다.

예시:

- 일정 결과
- 메일 결과
- 최근 결과

이 표면은 본문을 너무 밀어내지 않도록 기본은 한 줄 요약으로 유지하고, 자세한 내용은 row 클릭 또는 `세부 정보` 를 통해 확인하는 방향을 쓴다.

## 메모리란

메모리는 사용자의 선호, 프로젝트 상태, 반복되는 지시를 기억하는 기능이다.

현재 작업된 방향은 아래와 같다.

- 사용자 프로필
- 작업 요약
- 메모리 경계
- memory correction
- project memory

## 기억시키기

구조화된 입력으로 요청하면 가장 안정적으로 반영된다.

```text
기억해
내용: 나는 오전 회의를 선호해

기억해
내용: 앞으로 일정 요약은 짧게 알려줘
```

## 기억 수정하기

잘못 기억한 내용은 아래처럼 수정 요청을 할 수 있다.

예시:

```text
이건 기본 선호가 아님
수정할 기본 선호: 답변 길이는 기본적으로 간결하게 유지

잊어
내용: 지난번 임시 캘린더 테스트 메모
```

## 프로젝트 종료 표시

프로젝트나 작업이 끝났으면 아래처럼 말할 수 있다.

```text
이 프로젝트는 끝났어
프로젝트 메모: 지난번 캘린더 테스트 작업 종료
```

Nanobot 은 이를 바탕으로 memory correction 또는 task summary 를 정리할 수 있다.

## 주의사항

- 메모리 보정은 아직 모든 자연어를 완벽히 이해하는 단계는 아니다.
- 중요한 선호는 짧고 명확하게 말하는 것이 좋다.
- 민감한 개인정보는 저장 전에 꼭 필요 여부를 판단한다.

## 좋은 요청 예시

```text
기억해
내용: 일정은 가능하면 오후 2시 이후로 잡아줘

잊어
내용: 지난번 임시 캘린더 테스트 메모

이 프로젝트는 끝났어
프로젝트 메모: 지난번 캘린더 테스트 작업은 완료됐어
```

## 문제가 생겼을 때

- 기억이 잘못 반영된 것 같으면 바로 정정한다.
- WebUI 에서 memory correction prompt 가 보이면 내용을 확인한다.
- phrase 만 넣는 짧은 표현보다 `내용:` / `수정할 기본 선호:` / `프로젝트 메모:` 까지 함께 적는 편이 안전하다.
- 반복적으로 틀리면 어떤 문구가 잘못 저장됐는지 구체적으로 말한다.

더 깊은 배경은 [11. 심화 참고 문서 안내](11-advanced-reference.md) 의 observable reasoning / UX redesign 문서 묶음을 참고하면 된다.
