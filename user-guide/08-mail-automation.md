# 08. 메일 자동화 파일럿

## 메일 자동화란

Nanobot 의 메일 자동화는 n8n webhook 과 session metadata 를 이용해 메일 관련 작업을 도와주는 파일럿 기능이다.

현재 문서 기준으로는 calendar 보다 조심해서 다루는 pilot 단계로 보면 된다.

## 현재 목표

메일 자동화의 목표는 아래와 같다.

- 중요한 메일 확인
- 메일 요약
- 답장 초안 작성
- 전송 전 승인 흐름
- WebUI 에 action result 표시

## 사용자가 기대할 수 있는 일

자연어로 아래처럼 요청할 수 있다.

```text
중요한 메일이 있는지 확인해줘
오늘 처리해야 할 메일을 요약해줘
이 메일 답장 초안을 만들어줘
```

설정된 command 가 있는 경우 `/mail` 계열 명령으로도 접근할 수 있다.

예시:

```text
/mail
/mail status
```

## 승인 흐름

메일 전송은 실제 외부로 메시지가 나가는 작업이다.

따라서 안전한 운영 기준은 아래와 같다.

1. 먼저 메일 내용을 읽거나 요약한다.
2. 답장 초안을 만든다.
3. 사용자가 초안을 확인한다.
4. 전송 전 승인한다.
5. 승인 후 실제 전송한다.

## WebUI 표시

메일 자동화 결과는 WebUI 에서 아래처럼 보일 수 있다.

- action result card
- assistant details
- status summary
- Dashboard 요약

## 현재 한계

현재는 pilot backbone 이 정리된 상태로 이해하는 것이 안전하다.

- live credential 과 n8n webhook 상태에 따라 동작 범위가 달라진다.
- 모든 메일 경로가 end-to-end 로 항상 검증된 것은 아니다.
- 자동 전송보다는 draft 와 approval 중심으로 쓰는 것이 안전하다.

## 안전한 사용 예시

좋은 요청:

```text
중요한 메일만 요약해줘
답장 초안만 만들어줘. 아직 보내지는 마.
이 초안의 핵심만 3줄로 정리해줘
```

피해야 할 요청:

```text
확인 없이 전부 답장 보내줘
메일함 전체를 마음대로 정리해줘
```

## 추천 사용 흐름

1. 메일 상태를 확인한다.
2. 중요한 메일만 요약한다.
3. 답장 초안을 만든다.
4. 사용자가 직접 검토한다.
5. 승인 가능한 상태일 때만 전송한다.
